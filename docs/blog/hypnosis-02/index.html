Design

Every game in the world follows the same underlying logic. Every frame, something happens. This is called the ‘Game Loop’. Typically, three things happen every frame:

  1. Getting the user’s input
  2. Updating the game logic
  3. Rendering the game

It’s an easy way to break up the complexities of a game engine into modular parts.

An example of a simple game loop:

// This runs every frame
while( true )
{
    // Gets user input e.g. Keys pushed or mouse movement
    getInput( );

    // Does game logic e.g. Rotate 3D object or move player
    updateGame( );

    // Draws the game to the screen
    render( );
}

We will build off this simple premise to create a powerful game loop that our games will run on.

Main Function

I normally create a main function in Javascript which is the entry point for my program.
