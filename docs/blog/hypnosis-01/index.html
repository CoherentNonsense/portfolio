Hypnosis: A Small Game Engine for My Blog

Posted on December 15, 2020

A Game Engine for a Blog?

So to make my blog the special unicorn that it is, I want my content to have a fun and interactive side to it with tiny games and simulations that compliment my writing. These programs will be run using Javascript (The programming language used in browsers).

I don’t want to have to program everything from scratch everytime I want to make a small simulation. This is why I will be making a game engine which can do a bunch of things that all my programs will probably need like rendering sprites and getting user input.

Now, since this will be a rather large project, it’s going to need a rather cool name. And so I present to you: Hypnosis. I named it that because it’s what came up when I googled ‘cool word’.

Requirements

This is a list of things I need Hypnosis to do.

  • Rendering 2D sprites and 3D meshes
  • A 2D and 3D camera
  • Getting keyboard and mouse input
  • A simple 2D and 3D physics system
  • UI
  • Animations
  • Lots more

Now this is a lot of stuff. Like a lot of stuff. It would be stupid to try make all of this at once. So I have decided to make this framework very modular. This is the practice of developing code as small individual parts that can only do one thing. This will allow me to build the parts I need as I need them. For example, my first project will be rendering 3D objects using just math, so I can first work on the 3D rendering, 3D camera, and maybe a couple more things as I go along.

The Technology Used

Rendering

Rendering will be a big part of Hypnosis. There are two options when it comes to rendering on the web. Using the Javascript canvas and WebGl. The canvas uses your CPU while OpenGl uses your GPU. One of those acronyms has the word graphical. Surprisingly, it’s muuuuch faster at doing graphics but at the cost of complexity. For the sake of speed, I will be using the canvas which is fast enough for my purposes. And even so, do to the nature of modular code, it will be easy to switch out a canvas renderer for a OpenGl renderer later on if I need more power.
